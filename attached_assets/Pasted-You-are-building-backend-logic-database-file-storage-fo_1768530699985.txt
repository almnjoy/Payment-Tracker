DATABASE TABLES (CREATE THESE IN REPLIT SQL)

1) users_profile (bridge between Replit Auth and app data)
- user_id TEXT PRIMARY KEY                       // from Replit Auth
- role TEXT NOT NULL CHECK role IN ('admin','client')
- client_id TEXT NULL                             // FK -> clients.client_id, null for admin
- status TEXT NOT NULL DEFAULT 'active' CHECK status IN ('active','disabled')
- created_at TIMESTAMP DEFAULT NOW()
- updated_at TIMESTAMP DEFAULT NOW()

2) clients (business record, can exist before login)
- client_id TEXT PRIMARY KEY                      // readable IDs like CL-000001 or UUID-as-text
- display_name TEXT NOT NULL
- email TEXT
- phone TEXT
- notes TEXT
- created_at TIMESTAMP DEFAULT NOW()
- updated_at TIMESTAMP DEFAULT NOW()

3) leases (why they pay)
- lease_id TEXT PRIMARY KEY
- client_id TEXT NOT NULL                         // FK -> clients.client_id
- rent_amount_cents INTEGER NOT NULL
- due_day INTEGER NOT NULL                         // 1-28
- start_date DATE NOT NULL
- end_date DATE NULL
- status TEXT NOT NULL DEFAULT 'active' CHECK status IN ('active','ended','paused')
- created_at TIMESTAMP DEFAULT NOW()
- updated_at TIMESTAMP DEFAULT NOW()

4) invite_codes (invite-only onboarding)
- magic_number TEXT PRIMARY KEY                    // unique invite token
- client_id TEXT NOT NULL                          // FK -> clients.client_id
- lease_id TEXT NULL                               // FK -> leases.lease_id
- expires_at TIMESTAMP NOT NULL
- used_at TIMESTAMP NULL
- used_by_user_id TEXT NULL                        // FK -> users_profile.user_id
- created_by_user_id TEXT NOT NULL                 // FK -> users_profile.user_id (admin)
- created_at TIMESTAMP DEFAULT NOW()

5) invoices (bills owed)
- invoice_id TEXT PRIMARY KEY
- client_id TEXT NOT NULL                          // FK -> clients.client_id
- lease_id TEXT NULL                               // FK -> leases.lease_id
- title TEXT NOT NULL
- amount_cents INTEGER NOT NULL
- due_date DATE NOT NULL
- status TEXT NOT NULL DEFAULT 'open' CHECK status IN ('draft','sent','open','paid','overdue','void')
- stripe_invoice_id TEXT NULL
- stripe_hosted_invoice_url TEXT NULL
- created_at TIMESTAMP DEFAULT NOW()
- updated_at TIMESTAMP DEFAULT NOW()

6) payments (money received, supports manual now, Stripe later)
- payment_id TEXT PRIMARY KEY
- client_id TEXT NOT NULL                          // FK -> clients.client_id
- invoice_id TEXT NULL                             // FK -> invoices.invoice_id
- amount_cents INTEGER NOT NULL
- method TEXT NOT NULL CHECK method IN ('stripe','cash','venmo','zelle','bank_transfer','other')
- status TEXT NOT NULL DEFAULT 'paid' CHECK status IN ('pending','paid','failed','refunded')
- paid_at TIMESTAMP NULL
- stripe_payment_intent_id TEXT NULL
- stripe_charge_id TEXT NULL
- notes TEXT
- created_at TIMESTAMP DEFAULT NOW()
- updated_at TIMESTAMP DEFAULT NOW()

7) documents (metadata for files stored in App Storage)
- document_id TEXT PRIMARY KEY
- client_id TEXT NOT NULL                          // FK -> clients.client_id
- lease_id TEXT NULL                               // FK -> leases.lease_id
- invoice_id TEXT NULL                             // FK -> invoices.invoice_id
- title TEXT NOT NULL
- doc_type TEXT NOT NULL CHECK doc_type IN ('lease','receipt','notice','other')
- visibility TEXT NOT NULL DEFAULT 'client_and_admin' CHECK visibility IN ('client_and_admin','admin_only')
- storage_bucket TEXT NOT NULL                      // App Storage bucket name
- storage_key TEXT NOT NULL                         // object key/path inside bucket
- content_type TEXT NULL                            // ex: application/pdf
- file_size_bytes INTEGER NULL
- uploaded_by_user_id TEXT NOT NULL                 // FK -> users_profile.user_id
- created_at TIMESTAMP DEFAULT NOW()
- updated_at TIMESTAMP DEFAULT NOW()

8) external_accounts (placeholder for future Plaid/Stripe linking UI)
- account_id TEXT PRIMARY KEY
- provider TEXT NOT NULL                            // plaid, stripe, manual, mock
- nickname TEXT NOT NULL
- status TEXT NOT NULL                              // linked, needs_auth, error
- last_sync_at TIMESTAMP NULL
- created_by_user_id TEXT NOT NULL                  // admin user_id
- created_at TIMESTAMP DEFAULT NOW()
- updated_at TIMESTAMP DEFAULT NOW()

RELATIONSHIPS (MUST ENFORCE IN LOGIC)
- invite_codes.magic_number points to client_id and optional lease_id
- when user claims magic_number:
  - validate invite exists, not used, not expired
  - upsert users_profile for this Replit user_id:
      role='client', client_id=invite_codes.client_id
  - set invite_codes.used_at=NOW(), used_by_user_id=user_id
- clients -> leases -> invoices -> payments
- documents attach to a client and optional lease/invoice; actual file lives in App Storage

OBJECT STORAGE (REPLIT APP STORAGE) REQUIREMENTS
Implement a proper document upload and retrieval flow using Replit App Storage:
- Use one bucket for the app (ex: quickitprojects-docs) or environment-configured bucket
- Store objects using a predictable key pattern:
  - clients/{client_id}/leases/{lease_id}/{document_id}.pdf
  - clients/{client_id}/invoices/{invoice_id}/{document_id}.pdf
  - clients/{client_id}/other/{document_id}/{filename}
- Save storage_bucket and storage_key into documents table.
- Enforce access control:
  - Admin can upload, list, download all
  - Client can list/download only documents with visibility='client_and_admin' and matching their client_id
- For downloads, do NOT expose raw object storage credentials.
  - Provide a backend endpoint that streams the file to the browser, or generate a short-lived signed URL if supported.
- Support PDF first. Allow other types later.
