2) SERVER: Admin-only Plaid endpoints (in server/routes.ts)
Add these endpoints, all protected by isAuthenticated + isAdmin:

A) POST /api/admin/plaid/link-token
- Call plaid.linkTokenCreate:
  - client_name: "Quick IT Projects"
  - products: ["transactions"]
  - country_codes: ["US"]
  - language: "en"
  - user.client_user_id = current authenticated user id (the same id returned by /api/me)
Return: { link_token }

B) POST /api/admin/plaid/exchange
Body: { public_token, institution_id, institution_name }
- Exchange public_token for access_token + plaid_item_id using itemPublicTokenExchange
- Insert into plaid_items:
  - item_id = generatePlaidItemId()
  - admin_user_id = current user id
  - plaid_item_id, access_token
  - institution_id/name
- Fetch accounts via accountsGet(access_token)
- For each Plaid account:
  - Upsert into plaid_accounts for this item_id and plaid_account_id
  - Use accountId = generatePlaidAccountRowId() only when inserting a new row
  - Convert balances to cents (Math.round(balance * 100)) for current and available
- Initialize transactions via transactionsSync(access_token) with no cursor:
  - Upsert added transactions into plaid_transactions
  - Use transactionId = generatePlaidTransactionRowId() only when inserting new rows
  - Store raw_json as jsonb (store the Plaid txn object)
  - Store cursor + last_sync_at in plaid_cursors
Return: { item_id, institution_name }

C) POST /api/admin/plaid/sync
- For each plaid_items row for this admin_user_id:
  - Read cursor from plaid_cursors (may be null initially)
  - Call transactionsSync(access_token, cursor)
  - Upsert added + modified into plaid_transactions
  - For removed: delete rows where item_id + plaid_transaction_id match
  - Update cursor and last_sync_at
Return counts: { synced_items, added, modified, removed }

D) GET /api/admin/plaid/items
- Return list of linked items for this admin:
  - item_id, institution_name, status
  - last_sync_at (join from plaid_cursors)
Return array.

E) DELETE /api/admin/plaid/items/:itemId
- Ensure the item belongs to this admin_user_id
- Delete in order:
  - plaid_transactions where item_id
  - plaid_accounts where item_id
  - plaid_cursors where item_id
  - plaid_items where item_id
Return success.

F) GET /api/admin/plaid/account-summaries
- For this admin_user_id:
  - Query plaid_items + plaid_accounts + last_sync_at
- Return grouped payload by institution:
[
  {
    item_id,
    institution_name,
    last_sync_at,
    accounts: [
      { name, mask, type, subtype, current_balance_cents, available_balance_cents, iso_currency_code }
    ]
  }
]

IMPORTANT SECURITY:
- Never return access_token to client
- Never log secrets
